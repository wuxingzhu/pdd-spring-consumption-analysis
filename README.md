# 拼多多春季消费分析 —— 南昌 vs 深圳 双城对比

## 📊 项目简介

这是我的第二个数据分析项目，基于模拟生成的2026年春季（3-5月）拼多多消费数据，核心聚焦**南昌(二线) vs 深圳（一线）**双城对比。

项目完整覆盖了 **MySQL → Python → Tableau** 的数据分析全链路：
- 自主设计并生成带脏数据的模拟数据集
- Python 完成数据清洗与预处理
- MySQL 进行深度分析与复杂查询
- Tableau 构建交互式双城对比仪表盘

## 🔥 核心成果速览

| 指标 | 南昌 | 深圳 | 关键发现 |
|------|------|------|----------|
| 总消费额 | 127.5万 | 109.1万 | 南昌 +17% |
| 订单数 | 3,157 | 3,329 | 深圳 +5% |
| 客单价 | 404元 | 328元 | 南昌 +23% |
| 人均消费 | 1,256元 | 1,263元 | 基本持平 |
| 拼团率 | 39.5% | 39.8% | 高度一致 |

> 💡 **核心洞察**：南昌用户呈现“高质量、高客单”特征；深圳用户呈现“高频、活跃”特征。两城用户画像分化明显，但社交裂变行为高度趋同。

## 🔍 关键业务洞察

### 1. 执行摘要：双城画像分化明显

| 维度 | 南昌 | 深圳 | 解读 |
|------|------|------|------|
| 总消费额 | 127.5万 | 109.1万 | 南昌用户消费总额更高 |
| 订单数 | 3,157 | 3,329 | 深圳用户购买更频繁 |
| 客单价 | 404元 | 328元 | 南昌用户单次消费金额更高 |
| 人均消费 | 1,256元 | 1,263元 | 两城用户年度消费力相当 |

**结论**：南昌用户“买得少但买得贵”，呈现“高质量、高客单”特征；深圳用户“买得多但买得便宜”，呈现“高频、活跃”特征。

### 2. 品类偏好：服饰鞋帽是绝对主导

- 南昌：服饰鞋帽占比 **70%**，美妆个护 9%
- 深圳：服饰鞋帽占比 **60%**，美妆个护 12%
- **发现**：深圳用户在美妆个护上投入比例更高，可能反映一线城市对形象管理的更高关注

### 3. 价格带：50-100元是主力区间

- 双城 50-100元 订单占比均约 **42%**
- 100元以上订单：南昌 30% > 深圳 27%
- **发现**：南昌用户在高价位商品上的消费比例略高，与客单价更高的结论一致

### 4. 促销敏感度：双城高度一致

- 优惠券使用率：深圳 90.3% > 南昌 90.1%（几乎无差异）
- 平均折扣率：深圳 10.25% > 南昌 9.87%
- **发现**：拼多多用户整体价格敏感度高，双城差异极小

### 5. 社交裂变：拼多多模式的全国普适性

- 拼团订单占比：南昌 39.5%，深圳 39.8%
- 平均分享次数：南昌 2.76次，深圳 2.75次
- **发现**：社交裂变行为在两城没有显著差异，说明拼多多的“拼团+分享”模式已全国普及

### 6. 时间趋势：春季消费“双峰”特征

- **高峰1**（3月中旬/周11-12）：南昌春季大促，单周消费 **31万元**
- **高峰2**（4月中旬/周15-16）：深圳消费小高峰，达 **14.8万元**
- **发现**：5月下旬双城同步回升，可能是五一促销效应

### 7. 用户分层：高价值用户贡献主力

- 南昌：25%的高价值用户贡献了 **68%** 的消费
- 深圳：25%的高价值用户贡献了 **62%** 的消费
- **发现**：高价值用户人均消费南昌（4331元）> 深圳（3138元），进一步印证南昌用户的“高质量”特征

## 📝 项目复盘与思考

### 为什么要做这个项目？

完成第一个个人财务分析项目后，我发现自己用的技术栈是 Excel + PowerBI，但数据分析师岗位也一样常要求 MySQL + Python + Tableau。于是我想：能不能做一个项目，把这三个工具都用上？

选择“拼多多 + 春季 + 南昌vs深圳双城对比”这个方向，是因为：
1. **拼多多**的社交裂变和价格敏感特征很有意思
2. **南昌**是我实习的城市，**深圳**是我未来想去工作的城市
3. **双城对比**既有地域特色，又能体现商业分析能力

### 第一阶段：数据生成（没有数据集，就自己造）

拼多多公开数据集非常少。我决定：**自己生成一个模拟数据集**。

为了让它“真实”，我查了拼多多2025-2026年的消费报告，提炼出核心规律：女性用户占62%、58%订单在18-24点、拼团率约40%等。同时刻意制造了**空值、异常值、格式错误、重复数据、逻辑错误**等脏数据，模拟真实业务环境。

### 第二阶段：Python数据清洗

清洗过程包括：
- 处理空值（中位数填充、未知填充）
- 删除异常值（年龄>100、价格为负）
- 格式标准化（正则提取手机号、统一日期格式）
- 去重与逻辑修正（拼团订单但分享次数为0 → 修正为1）

最终从9880行原始订单清洗到7207行有效订单。

### 第三阶段：MySQL深度分析

编写了8个核心SQL查询，涵盖双城画像、品类偏好、价格带、促销敏感度、社交裂变、时间趋势、用户分层、关联购买等维度。过程中遇到版本兼容问题（PERCENTILE_CONT不支持），改用NTILE(4)解决。

### 第四阶段：Tableau可视化

设计4页面仪表盘，确立“南昌青绿 + 深圳暖橙”配色方案，采用浮动布局精细控制排版。通过发布到Tableau Public再打印为PDF的方式，解决了导出难题。

### 一点小感悟

这个项目让我深刻体会到：**没有数据就创造数据，脏数据不是麻烦而是机会，工具永远是为问题服务的。**

从数据生成、清洗、分析到可视化，完整走完一遍后，我对数据分析的理解不再是零散的“我会用某个工具”，而是形成了完整的解决问题的方法论。
